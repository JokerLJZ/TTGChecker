# 角色设定
你是一个经验丰富的 Python 自动化测试工程师和爬虫专家。

# 任务目标
请帮我编写一个基于 Python 的自动化脚本，用于实现 PT 网站 TTG (Totheglory) 的每日自动签到功能。

# 技术栈要求
- 语言：Python 3.10+
- 自动化框架：建议使用 Playwright（或者 Selenium），需能够接管本地已打开的 Firefox 浏览器，或通过加载指定的用户配置文件（Profile）来免登录保持 Cookie 状态。
- 通知服务：WxPusher API (通过 HTTP 请求发送)
- 运行环境：debian 13

# 已确认实现方案
- 采用 `Playwright + Firefox Persistent Profile` 作为主实现路径。
- 不直接依赖“接管当前已打开的 Firefox 进程”，而是通过加载指定 Firefox Profile 目录复用登录态；这是当前实现中更稳定、可部署的方案。
- 配置文件采用 `JSON`，主入口为 `main.py`。
- 项目按模块拆分为：配置读取、浏览器操作、状态记录、通知发送、流程编排。

# 核心业务与场景需求
1. **浏览器环境适配**：
   - 目标浏览器为 Firefox。
   - 已知问题：启动 Firefox 时会强制打开一个无法关闭的默认页面。
   - 解决策略：脚本启动后，不要尝试关闭第一个标签页，而是直接新建一个标签页（New Tab）并在新标签页中打开 TTG 的网站进行后续操作。
2. **反爬虫绕过与操作模拟**：
   - TTG 有一定的反爬机制，不能仅发送 HTTP Requests。
   - 必须等待页面完全加载，并找到签到按钮所在的 DOM 元素。
   - 必须使用模拟人类行为的真实鼠标点击动作（例如 Playwright 的 `.click()` 或 Selenium 的 ActionChains）来触发签到。操作前后加入适量的随机等待时间（1-3秒）。
3. **状态记录与补签机制（漏签检测）**：
   - 在本地维护一个简单的状态文件（如 `checkin_log.json` 或 `sqlite`），记录每次成功签到的日期。
   - 脚本每次运行时需检查当天的签到状态。如果发现前一天或当天设定的时间段内未执行成功（即存在漏签），应立即启动补签/重试流程。
   - 这里的“补签”按工程实现定义为：检测到昨天或今天漏签后，立即触发当前时刻的重试，并在通知中带出漏签日期。若 TTG 站点本身不支持历史日期补签，则脚本不承诺回补历史签到记录。
4. **WxPusher 消息通知**：
   - **签到成功**：抓取页面上的成功提示信息（如获取到的魔力值等），通过 WxPusher 发送给指定 UID。
   - **异常报警**：如果出现“页面无法正常打开（如超时、502、404）”、“找不到签到按钮”或“账号未登录”等异常情况，捕获异常栈或截图，并通过 WxPusher 发送紧急报警通知。

# 代码输出要求
1. 请提供完整的、模块化的 Python 代码（分为配置读取、浏览器操作、通知发送、状态管理等模块）。
2. 提供详细的注释，解释如何获取 Firefox 的 Profile 路径并在代码中配置。

4. 考虑到代码可能运行在有头（GUI）或无头（Headless）环境中，请提供一个易于切换 headless 模式的变量配置。
